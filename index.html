Manga Downloader

Paste a WeebCentral link, pick chapters, get a ZIP or PDF. Runs entirely in your browser — nothing is uploaded anywhere.

This site talks to WeebCentral directly from your browser. That only works if WeebCentral allows it (a setting called CORS). If it doesn't, this app automatically routes the request through a free public proxy instead — but those are shared, rate-limited, and sometimes just offline. That's the most common reason a fetch fails.

Try this first

  1. Just hit Fetch again — public proxies often fail for a few seconds, then work.
  2. Wait 30–60 seconds if it keeps failing (you may be rate-limited).
  3. Open the Advanced log below the progress bar — it shows exactly which step and which proxy failed, which helps if you report a bug.

Reliable fix: use your own free proxy

Free public proxies get overloaded. A personal Cloudflare Worker is your own private one — nobody else uses it, so it doesn't get rate-limited by other people's traffic. It's free and takes about 5 minutes the first time.

1
Open Cloudflare Workers and sign in

Go to the link below and sign in, or create a free account if you don't have one yet.

https://workers.cloudflare.com/
Open it in a new tab →
2
Set up your workers.dev subdomain

Every Cloudflare account needs a one-time "subdomain" before it can run any Worker — this is the something.workers.dev part of your future Worker's address.

The first time you open Workers & Pages in the sidebar, Cloudflare should prompt you to pick one automatically. Type anything (a nickname, your name, random letters — it doesn't matter) and confirm. If you don't see that prompt, it usually means it was already set up silently — you can just move on to Step 3.

Can't find it, or the page looks stuck? Cloudflare's dashboard has a built-in AI assistant — look for an "Ask AI" button near the top of the page. Tap it, then paste this prompt: 
Create my workers.dev subdomain for this account. Pick an available name if needed and finish the setup so I can deploy my first Cloudflare Worker.
It'll walk you through it directly, no digging through menus required.
3
Create the Worker
  1. In the sidebar, tap Workers & Pages
  2. Tap the blue Create application button
  3. Choose Create Worker — not a template, not "Import a repository"
  4. You'll see a random name like calm-tree-6ce3 — leave it, or rename it to whatever you like
  5. Tap Deploy
This first deploy publishes a placeholder "Hello World" script — that's expected. You'll replace it with the real code in the next step.
4
Replace the code and deploy again
  1. After it deploys, tap Continue to project (or open the Worker from your Workers & Pages list)
  2. Tap Edit code
  3. Select everything in the editor and delete it
  4. Paste in the code below
  5. Tap Deploy again — this pushes your code live
